Japan Post wizard Step 3 (item phase, mode=item):
  Order in script: optional ship type click/select -> step3CategorySales -> step3ContentLine
    -> itemTitle -> originCountryCode -> weightGrams -> hsCode -> unitPriceJpy -> quantity
    -> sizes + VAT fields -> step3OpinionConfirm checkbox
  Fill selectors in playwright_selectors_japanpost.sample.json:
    clicks: step3ShippingEpacketLite, step3ShippingEMS, step3CategorySales, step3ContentLine, step3OpinionConfirm
    fields: step3ShipmentTypeSelect (with label.jpShipmentSelectValue or shippingService), itemOriginCountry, unitPriceJpy
  label_input.json: originCountryCode, unitPriceJpy, jpShipmentSelectValue (exact <option value> for shipment type select)

Japan Post wizard Step 4 (mode=step4extras in fillPhases):
  Order in script: step4FlightWeightGrams (or billableWeightGrams / weightGrams fallback)
    -> clicks.step4AgreementCheckbox -> clicks.step4ExpandOtherShippingInfo
    -> taxCode, vatNumber, iossNumber, rutNumber, referenceNumber (eBay / EU などで必要な分だけ label_input に入れる)
  Selectors: fields.step4FlightWeightGrams, fields.taxCode, fields.referenceNumber + the three clicks above.
  同意チェックは Playwright の check() でオンにします（クリックのみより安定）。

fillPhases の clickAfter（任意）:
  各フェーズの入力のあと、selectors.clicks のキーを順にクリックします（例: item 後に step3Next、confirm 後に confirmNext）。
  セレクタが空なら何もしません。全自動に近づけるときは jp:dump で得た「次へ」ボタンの CSS を step3Next 等に記入。

JP_WIZARD_CLICK_FALLBACK=1（任意・実験用）:
  セレクタが空のとき、文言で「次へ」「内容確認」などを推測クリックします。誤クリックしうるため本番はセレクタ推奨。
  PowerShell 例: $env:JP_WIZARD_CLICK_FALLBACK="1"; npm run jp

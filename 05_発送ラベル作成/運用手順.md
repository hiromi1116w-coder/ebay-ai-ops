# 発送ラベル作成 班（最優先で自動化）

## 目標（ロードマップ Phase 1）

**ラベルの「作成」までは完全自動**を目指す。

- **完全自動の意味**：必須情報が揃った案件について、人の承認なしで **ラベル用ファイル（PDF 等）の生成まで** 一連で行う。郵便局／CPass は **チェックボックス等で明示選択**（将来、ルールに基づく自動振り分けに拡張してもよい）。
- **人の手が入るのは主に**：印刷・ラベル貼付、例外案件の判断、入力データの訂正（住所不備など）。
- **印刷**：プリンタやOSの都合で「印刷ボタン」まで含めるかは後から決められる。**まずはファイル生成までを100%自動**に寄せる。

## 処理の順番（UI・データの流れ・確定案）

次の順で作ると、入力がブレずにラベル生成までつながりやすい。

1. **キャリアの選択**  
   「郵便局で発送」「CPass で発送」を切り替える **チェックボックス（またはラジオ）** を用意する。1注文・1梱包単位で選ぶ。

2. **商品（SKU）ごとの梱包情報 — 1注文・複数行**  
   一度の注文で **2品以上を出すことがある**ため、重量・サイズは **行を複数** 持てるようにする（品目／ラインごとに1行が基本）。

   **1梱包にまとめて出す場合**  
   - **重量**：各行の **合計** をラベル用の総重量とする。  
   - **サイズ**：キャリアが求めるのは **外装1箱の寸法** なので、実測した **梱包後の1組の縦×横×高** を入れる。  
     - 運用は次のどちらかに **固定** する：  
       - **A**：品目行は重量だけ入力し、最後に **「外装箱」1行** で寸法だけ入れる。  
       - **B**：最初の行に **合算重量＋外装箱寸法** をまとめて書き、他行は重量のみ（参照用）。

   **分納（複数梱包）の場合**  
   - 行に **梱包番号（1, 2, …）** を付け、**梱包ごとに重量・寸法をまとめた行** を作る。ラベルは **梱包番号ごとに1枚** 生成する。

3. **eBay 側データの取得とマージ → ラベル生成**  
   上記に加え、**eBay のお届け先**、国・地域によって必要な **VAT／IOSS／EORI、RUT（チリ）** などの **税・通関用識別子** を取得し、選択したキャリア用フォーマットで **ラベルファイルを生成**する。

**補足**：VAT や RUT は、注文・バイヤー入力・eBay の項目の有無で **空になることがある**。その場合は **生成せず「要確認」** に回す（手入力フィールドを用意してもよい）。

## 完全自動で守るガードレール

- 入力直後に **自動バリデーション**（必須項目、重量・寸法の上限、宛先フォーマット、サービス可否）。失敗したら **生成せず** キューに「要確認」として残す。
- 成功・失敗とも **ログ必須**（日時・注文ID・キャリア・出力パス・エラー理由）。
- **キルスイッチ**：異常検知時は新規生成を止められるようにする。

## 入力（最低限そろえる項目）

- 注文ID（または Sellsta／eBay 上の参照ID）
- 送り先（または確定済みラベル用データの出どころ）
- **行ごと**：SKU または明細ID、重量、サイズ（梱包単位に応じて上記ルールで入力）
- 発送サービス（例：追跡の有無、国際／国内）
- 分納のときは **梱包番号**（1梱包のときは省略可）

## 出力

- 印刷用ラベル（PDF 等、運用で決める形式）
- ログ：日時・注文ID・使用キャリア・ファイルパスまたは印刷済みフラグ

## 例外（最初は自動の対象外にしてよい）

- 住所不備・税関・同梱の判断が必要な案件
- フォーマットや料金が通常と異なる特殊梱包

## 承認ポイント（移行時のみ）

- ルールやフォーマットを変えた直後は、**サンプル数件で出力ファイルを人が目視確認**し、問題なければ **承認なしフル自動** に戻す。

## 現行業務フロー（ユーザー実務から確定）

1. eBay で注文と送る商品を確認する。  
2. 商品の重量・サイズを測る。  
3. eBay のお届け先をコピーする。  
4. 郵便局サイトにログインし、宛先情報を貼り付ける。  
5. 発送方法を選ぶ。通常は **eパケットライト**。  
6. ただし、**重量が2kg超** または **サイズが90サイズ超** の場合は **EMS** を選ぶ。  
7. 内容品種別は **販売品** を選ぶ。  
8. 商品タイトルを入力する。  
9. 原産国は **JAPAN（日本）**（デフォルト想定）。  
10. 重量を入力する。  
11. HSコードを調べて入力する。  
12. 商品価格・個数（通常1）を入力する。  
13. 危険物確認のチェックを入れて次へ進む。  
14. 再度重量を入力する。  
15. 有償/無償は **有償** を選ぶ。  
16. 「その他の発送情報」で、**TAXコード/VAT番号/備考** に VAT や RUT 情報を入力する。  
17. 内容確認後に確定し、通知メールを受け取る。  

## 自動化仕様（Phase 1: 郵便局のみ）

- 注文は **eBayから自動取得**（未発送・処理対象）。  
- 取得した注文を画面に並べ、発送班が以下だけ入力/選択する。  
  - 郵便局 or CPass（現段階は郵便局固定でも可）  
  - SKU行ごとの重量  
  - 外装箱のサイズ  
  - 商品タイトル/HSコード（未取得時のみ）  
- システムが eBay 情報（宛先、VAT/IOSS/RUT など）と入力値をマージし、郵便局フォーム入力を自動実行する。  
- 条件分岐は自動で適用する。  
  - 通常: eパケットライト  
  - 2kg超 または 90サイズ超: EMS  
- ラベル作成確定まで自動で進め、結果（成功/失敗）をログに残す。  

## 郵便局スクリプト（Playwright・先に完成させる用）

1. `label_input.json` で **`carrier` を `JapanPost`** にし、`weightGrams` と `sizeCm`（例 `20x10x10`）を実測で入れる。  
2. 必要なら `shippingService` で上書き（`ePacketLite` / `EMS` を含む文字列。未指定なら **重量2kg以下かつ3辺合計90cm以下 → eパケットライト、それ以外 → EMS** とコンソールに推奨表示）。  
3. ターミナルで `05_発送ラベル作成` に移動し、`node .\autofill_japanpost_playwright.mjs` を実行。  
4. ログイン後、ラベル入力画面まで進めて **Enter** → 自動入力 → コンソールの推奨に従い **eパケットライト or EMS を手動で選ぶ**（`playwright_selectors_japanpost.sample.json` の `serviceEpacketLite` / `serviceEMS` にセレクタを入れるとクリックまで自動化）。  
5. 確認後 **Enter** → スクリーンショットは `autofill_japanpost_preview.png`。  
6. 画面上で確定・ラベル発行まで進める。  

**セレクタ**: 実画面の入力欄に合わせて `playwright_selectors_japanpost.sample.json` の `fields` / `clicks` を更新する（仮の `#receiverName` 等のままでは入力されないことがある）。

## 完成系ルール（2026-05-07 確定）

- 対象キャリアは **郵便局 / CPass / 特殊（それ以外）** の3区分とし、注文ごとに選択する。  
- キャリア選択後は配送サービスも必須で選択する。  
  - 郵便局: `ePacketLite` / `EMS`  
  - CPASS: `CPASS Economy` / `CPASS DHL` / `CPASS FedEx`  
- 入力は「重量」「サイズ」を必須とし、複数商品は注文内の複数行で扱う。  
- 宛先氏名・住所・商品タイトル・税情報（VAT/IOSS/RUT 等）は **eBay をマスタ** として自動取得する。  
- ラベル文字列は ASCII ベースに正規化し、特殊文字は自動変換する（例: `ß -> ss`）。  
- 郵便局/CPass の文字数制限を超える場合は、氏名・住所・タイトルを自動短縮する。  
- 商品タイトルは長文時に省略し、原則として末尾に `NO battery` を付与する。  
- HSコードは商品情報から自動推定し、推定不可または低信頼時のみ手動確認に回す。  
- 変換前後の値（タイトル、氏名、住所、税情報、HSコード）をログに残し、追跡可能にする。  
- 必須データ不足・形式不正時はラベルを生成せず、対象注文を「要確認キュー」に送る。  

## 配送オプションごとの選択ルール（運用確定）

### 1) 郵便局

- `ePacketLite` を優先選択する条件  
  - 重量が 2kg 以下  
  - サイズが 90 サイズ以下  
  - 危険物に該当しない  
- `EMS` を選択する条件  
  - 重量が 2kg 超  
  - またはサイズが 90 サイズ超  
  - または `ePacketLite` 対象外国・対象外条件に該当

### 2) CPASS

- 選択肢は次の4種類（画面表示名ベース）  
  - `eBay SpeedPAK Economy`  
  - `eBay SpeedPAK - Ship via DHL`  
  - `eBay SpeedPAK - Ship Via FedEx @International Connect Plus`  
  - `eBay SpeedPAK - Ship Via FedEx @International Priority`
- 基本ルール  
  - `eBay SpeedPAK Economy` が表示される場合は **Economy を優先選択**  
  - Economy が表示されない場合は、表示されている 3方式（FedEx 2種 + DHL）から選択
- Economy 非表示時の選択ルール  
  - まず見積り価格の最安値を選ぶ  
  - 同額なら優先順は `FedEx @International Connect Plus` -> `FedEx @International Priority` -> `DHL`
- 画面フロー（CPASS の実順）  
  1. `発送手続き待ち` ページで対象アイテムを選択  
  2. `発送手続き` タブで対象アイテムの `編集` を開く  
  3. **先に** HSコード（DHL/FedEx のときのみ・10桁数字）・重量・寸法を入力する  
  4. **その後** 配送方法の **4択**（Economy / FedEx / DHL）から選ぶ  
  5. 保存し、送料・ラベルへ進む
- HSコードルール  
  - `eBay SpeedPAK Economy` は HSコード不要  
  - `DHL` / `FedEx` は HSコード必須（**数字10桁のみ**、`.` や記号は不可）
- バッテリールール  
  - 商品にバッテリーが含まれる場合は `バッテリーあり` をチェック  
  - バッテリー種別も必須入力
- **重量・寸法（必ず確認）**  
  - CPASS は **重量・サイズ（容積重量含む）** が料金・可否に直結する。**実測値**を入れる（デモ用の仮値のまま本番確定しない）。  
- **コストを最優先しない場合**  
  - Economy ではなく、**HSコード（10桁数字のみ）を入れたうえで FedEx または DHL を選ぶ**（追跡・納期・通関まわりを優先する運用）。

### 3) 例外と停止条件

- `carrier` と `shippingService` の組み合わせが不正な場合は自動停止  
  - 例: `carrier=JapanPost` で `CPASS DHL` を選択
- 重量・サイズ未入力時はサービス判定を行わず `要確認キュー` へ送る
- 仕向国のサービス対象外は自動で代替候補を提示し、確定は手動確認

# 郵便局 / CPASS 自動入力フロー設計（実装用）

## 目的

`label_input.json` を入力データ源にし、以下を自動化する。

1. 対象サイトへログイン
2. ラベル作成画面へ遷移
3. 各入力欄へ自動入力
4. 作成（確定直前または確定まで）
5. 結果ログ保存

---

## 実装方針（段階）

### Phase A: 半自動（先に安定化）
- ログインは人が実施
- ログイン後の画面操作を自動化
- 確定ボタンは最初は手動（誤発行防止）

### Phase B: 自動ログイン追加
- ID/PW を安全保管（Windows Credential Manager 等）
- ログイン処理を自動化
- 2FA 発生時は「手動介入待ち」にする

### Phase C: 完全運用
- 注文キューから順次処理
- 失敗時リトライ、要確認キュー振り分け
- 監査ログとスクリーンショット保存

  - **CPASS 自動入力スクリプト**: `shippingService` が**空または未設定**のときは **Economy 扱い**（HS不要）。FedEx/DHL で HS を自動入力させるときは `shippingService` に `dhl` または `fedex` を含む文字列を必ず入れる。
- `cpassStage` (`paid_list` / `edit_form`)
- `buyerName`
- `postalCode`
- `countryCode`
- `stateOrProv`
- `city`
- `addressLine1`
- `addressLine2`
- `phone`
- `itemTitle`（`NO battery` 付与済み）
- `quantity`
- `vatNumber` / `iossNumber` / `rutNumber`
- `hsCode`
- `hsCodeRequired`
- `hasBattery`
- `batteryType`
- `weightGrams`

## 変換ルール（共通）

- 非ASCII文字は変換（例: `ß -> ss`）
- 項目ごとの最大文字数を超える場合は切り詰め
- 空欄不可の項目が空の場合は「要確認キュー」へ
- `carrier` と `shippingService` の組み合わせ不正は処理しない（例: carrier=JapanPost で CPASS DHL を選んだ場合）
- 税情報は eBay をマスタにし、無い場合は空欄 + 要確認フラグ

## サービス選択ルール（実装用）

### JapanPost

- デフォルト: `ePacketLite`
- 以下のいずれかで `EMS` に切替
  - `weightGrams > 2000`
  - `sizeCm` が 90 サイズ超
  - 仕向国または品目条件で ePacketLite 不可

### CPASS

- 対応サービス（画面表示名）
  - `eBay SpeedPAK Economy`
  - `eBay SpeedPAK - Ship via DHL`
  - `eBay SpeedPAK - Ship Via FedEx @International Connect Plus`
  - `eBay SpeedPAK - Ship Via FedEx @International Priority`
- 選択ロジック
  1. Economy が表示される場合は Economy を選択
  2. Economy が表示されない場合は、残り3方式の見積り価格を比較して最安値を選択
  3. 同額時は次の優先順で選択  
     `FedEx @International Connect Plus` -> `FedEx @International Priority` -> `DHL`
- 2段階運用
  - `paid_list` フェーズ: 対象アイテム選択（手動）
  - `edit_form` フェーズ: 編集画面で自動入力
- **CPASS の実務順（重要）**  
  1. 編集画面で **HS・重量・寸法を先に**入れる（DHL/FedEx 時は HS 必須）  
  2. **そのあと** 配送方法の **4択** を選ぶ  
  - 自動入力スクリプトもこの順で動き、4択が見えるまで **追加の Enter 待ち** を挟む
- HSコード
  - Economy: 不要
  - DHL/FedEx: 必須（10桁数字のみ）
- **重量・寸法（検証必須）**  
  - CPASS では **実測の重量・外装箱寸法** を正しく渡す（仮値は検証・デモ専用）。容積重量で料金が変わるため、寸法の誤りは直接ミスコストにつながる。  
- **コスト最優先でないとき**  
  - `shippingService` を Economy ではなく **FedEx または DHL** にし、`hsCode` を **10桁数字**で必ず入れる（追跡・納期・通関を優先するフロー）。  
- バッテリー
  - `hasBattery=true` の場合、バッテリーチェックと `batteryType` 入力を実施

### 検証

- `shippingService` が空欄なら実行しない
- `carrier` と `shippingService` の整合表で必ず検証する

## 例外処理ルール

- 2FA/CAPTCHA検出時: 自動停止し、手動介入待ち
- 入力項目不足: 発行せず `needs_review` に移動
- 画面遷移失敗: リトライ 2回まで
- 発行成功時: 注文ID、キャリア、時刻、結果URLをログ保存

### 実行メモ

1. Node.js をインストール
2. `npm init -y`
3. `npm i playwright`
4. セレクタJSONのURL/selectorを実画面に合わせて更新
5. `node autofill_japanpost_playwright.mjs`

### 郵便局スクリプト更新（優先完成用）

- `weightGrams`・`sizeCm` から **eパケットライト / EMS** を推奨表示（2kg超または3辺合計90cm超なら EMS）。  
- `shippingService` に `epacket` / `ems` が含まれれば推奨より優先。  
- `serviceEpacketLite` / `serviceEMS` にセレクタを入れると **該当ラジオ・ボタンを自動クリック**。未設定ならコンソール推奡のうえ **手動選択**。  
- スクリーンショット出力: `autofill_japanpost_preview.png`（CPASS の `autofill_cpass_preview.png` と分離）。  
- `carrier` が `CPASS` のときは警告ログのみ（郵便局試験時は `JapanPost` にすること）。

### CPASS 追加メモ

- `autofill_cpass_playwright.mjs` を追加済み
- `playwright_selectors_cpass.sample.json` のURL/selectorを実画面に合わせて更新
- 実行コマンド: `node autofill_cpass_playwright.mjs`
- サービス自動選択ロジックを実装済み  
  - Economy が表示される場合は Economy を選択  
  - Economy がない場合は DHL/FedEx 候補の最安値を選択（同額時は FedEx Connect Plus -> FedEx Priority -> DHL）  
  - `shippingService` に `dhl` / `fedex` が含まれるときは **Economy を選ばず**、該当行の **「選択」** をテキスト一致でクリックするフォールバックあり

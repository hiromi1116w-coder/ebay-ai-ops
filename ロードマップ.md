# 今後のロードマップ（自動化・半自動化の優先順位）

## 確定した優先順位（この順で育てる）

1. **05 発送ラベル作成**（郵便局／CPass）— **いま一番先に進める**  
   必要情報が揃った案件は **ラベルファイル（PDF 等）の生成まで完全自動**（人の承認なし）。印刷・貼付は別。担当：**発送班**。
2. **02 価格調整** — 最安確認・利益試算・変更案の一覧化 → 反映はルールと検証に応じて人または自動へ。
3. **04 在庫管理**（スプレッドシート）— 差分レポート・要注意SKU。列定義は必要に応じて固定。
4. **01 セルスタ出品** — チェックリスト・下書き支援 → 条件が整えば公開まで自動化（`共通ルール/運用ポリシー.md` の段階移行）。
5. **03 Gmail（自動返信）** — **最後**。メール担当（班）と **自動送信OK一覧** を先に作ってから限定自動送信へ。

---

## 優先順位の決め方（共通ルール）

次の4つで並べ替える（班の中でもタスクの順番を決めるときに使う）。

| 観点 | 意味 |
|------|------|
| **効果** | 時間削減・ミス削減が大きいか |
| **リスク** | 誤操作で売上・評価・ポリシー違反につながりやすいか |
| **工数** | ルール化・ツール化までの手間 |
| **依存** | 他班（データ・手順）が先に必要か |

原則：**リスクが高い操作は「下書き・提案」までにし、確定は人**（移行条件は `共通ルール/運用ポリシー.md`）。

## フェーズの進め方

### Phase 0 — 土台（並行して薄く維持）

- 各班の `運用手順.md` に **入力・出力・承認ポイント** を1枚で書く。
- `作業ログ` に、自動化の試行と失敗例を残す。

### Phase 1 — 発送ラベル（最優先）

- **目標**：入力（重量・サイズ・送り先・必要なら追跡／サービス種別）→ **ラベル用ファイル生成まで完全自動**（バリデーション通過時のみ）。印刷はプリンタ連携の都合で任意。
- **例外**：同梱、住所不明、サービス制限、特殊梱包は **生成しない** でキューに残し、人が処理する。
- **ログ**：いつ・どの注文ID・どのフォーマットで出したか（失敗理由含む）を残す。

### Phase 2 — 価格調整

- Sellsta／eBayの最安確認 → 利益試算（還付金は別ルールどおり）→ **変更案一覧**。反映は検証後に半自動→自動へ。

### Phase 3 — 在庫管理

- eBay在庫と実在庫の差分、ホットSKUの監視。スプレッドシート列が固まったらレポートを定型化。

### Phase 4 — セルスタ出品

- 出品前チェックリスト、属性・タイトルの下書き。公開の自動化はチェックリスト合格＋ログ＋キルスイッチを前提にする。

### Phase 5 — Gmail 自動返信

- 担当班・**自動送信OK一覧**・禁止ワード・送信ログを先に。クレーム・返金・個人情報は人のみ。

## メモ（優先を変える条件）

- ラベル周りで **仕様変更・ログイン方式・印刷制限** が重い場合は、Phase 1 の中で「まず半自動（ワンクリック生成）」に落としてから全自動へ伸ばす。
- PC負荷で価格見直しが週1できないなら、**全商品は月1＋ホットSKUは毎日**（プロフィールの方針どおり）。
- eBay／Sellsta／郵便局／CPassの仕様変更があったら、**自動範囲をいったん縮めて** 手順とログからやり直す。

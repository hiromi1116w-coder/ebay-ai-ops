# 02 価格調整 — 自動化ロードマップ

## ゴール

セルスタ（Sellsta）で管理している eBay 出品について、次のサイクルを **再現可能・監査可能** にする。

1. 競合の最安値（**商品価格＋送料の合計**で比較 — 運用で確定済み）を取得
2. **最安 − 0.01 USD** を候補価格とする
3. その価格での **利益率が 1% 以上**（還付金は判定に含めない）なら変更案を出す
4. 利益が足りない場合は **据え置き／値上げ案** を出す（無理な値下げはしない）
5. 最初は **変更案一覧のみ**。検証後にセルスタ／eBay への反映を半自動→自動へ

親ロードマップ（`ロードマップ.md`）の **Phase 2** に対応。`05_発送ラベル作成` と同様、**Phase A 半自動 → Phase B ログイン自動 → Phase C キュー運用** で育てる。

---

## セルスタ標準機能との関係（最初に確認すること）

[セルスタ応用設定](https://sellsta.biz/application.html) には次がある。

- 利益率に基づく自動価格調整（仕入変動に追従）
- 売れ残りの自動値下げ
- 再販価格の自動設定

今回の運用は **「eBay 上の競合最安に対して 0.01 USD 安く張る」** が中心。セルスタ標準がこのロジックを満たすか、**1 回だけ設定画面で確認**する。

| 結果 | 方針 |
|------|------|
| 標準機能で足りる | 本リポジトリは **利益試算の検証・ログ・例外 SKU** に特化し、ブラウザ自動化は最小限 |
| 足りない／細かい制御が必要 | 本ロードマップどおり **自前ツール＋Playwright** を進める |

## フェーズ一覧

### Phase 0 — ルールとデータの固定（1〜2 日）

**成果物**

- `運用手順.md` の週次／月次サイクル確定
- `profit_rules.json`（手元用・git には sample のみ）で利益計算式を固定
- 対象 SKU リストの出どころ（セルスタ一覧のエクスポート or 手動 CSV）を決める

**やること**

- 「最安値」の定義：**価格 + 送料の合計**（確定）
- **利益・手数料**：セルスタの利益計算ツールをマスタとする。リポジトリの試算は補助。1 件ずつセルスタと突合してからスケールする
- 還付金は試算に入れない（`00_セラープロフィール/seller_profile.md` どおり）
- ホット SKU（毎日）と全体（週 1 or 月 1）のリスト運用
- **セルスタ仕様の確認**（応用設定・一覧・利益画面・API/エクスポートの有無）をスクショ付きで記録 → 自動化の分担表を更新
- 将来 **1 日 ~100 件** を見据え、1 セッションあたりの上限・待機秒・失敗時キューを設計（セルスタ側制限が分かり次第数値化）

---

### Phase 1 — オフライン利益試算（提案 CSV のみ）

**成果物**

- `lib/profit_calc.mjs` / `lib/price_proposal.mjs`
- `run_price_proposal.ps1` → `output/price_proposals_YYYYMMDD.csv`
- 入力：`listings_input.json`（手入力 or セルスタからコピーした最安・現価・仕入）

**やること**

- 1 SKU 手入力で **提案価格・利益率・action（lower / raise / hold / skip）** が期待どおりか検証
- 10〜20 SKU でセルスタ上の利益表示と突合し、計算式を微調整

**自動化範囲**：価格の **変更はしない**。人が CSV を見てセルスタで反映。

---

### Phase 2 — 最安値の半自動取得（Playwright Phase A）

**成果物**

- `autofill_sellsta_price_scan_playwright.mjs`（雛形）
- `playwright_selectors_sellsta.sample.json`
- `run_sellsta_scan.ps1`
- ログインは **人**、ログイン後に SKU 一覧から eBay 競合画面まで遷移し **最安値をスクレイプ or 画面ダンプ**

**やること**

- セルスタの「eBay で見る」導線の DOM を 1 商品で確定（`run_dump.mjs` パターンを流用）
- 取得失敗・CAPTCHA・2FA は **停止して手動**（`05_発送ラベル作成` と同じ）
- 取得結果を `listings_input.json` にマージするスクリプト

**リスク対策**：1 セッションあたりの SKU 数上限、リクエスト間隔（例 3〜5 秒）、スクリーンショット保存

---

### Phase 3 — 一括提案パイプライン（まだ反映は人）

**成果物**

- `run_pricing_cycle.ps1`：scan → merge → propose → CSV + `作業ログ` 追記
- `operator_mustdo.mjs`：要確認 SKU のチェックリスト出力

**やること**

- 週次バッチで全 SKU（またはホットのみ日次）を回す
- `skip` 理由を必ず CSV に残す（競合なし、最安取得失敗、利益不足で floor など）

---

### Phase 4 — 半自動反映（Sellsta 上で価格入力まで）

**成果物**

- `apply_price_proposals_playwright.mjs`：提案 CSV を読み、セルスタの価格欄へ入力
- **確定ボタンは最初は手動**（`共通ルール/運用ポリシー.md`）

**ガードレール**

- 1 回の実行で変更できる最大件数
- 提案と現価の差が **X% 超** なら `needs_review` に回す
- 実行前後のスクリーンショット

---

### Phase 5 — 条件付き完全自動（任意・十分な検証後）

- チェックリスト合格 SKU のみ自動反映
- キルスイッチ・日次サンプリング監査
- 新カテゴリ・高額 SKU は人の枠を残す

**不変**：還付金を含めた「税務上の最終判断」、アカウント操作は人のみ。

## 技術スタック（05 発送ラベルと揃える）

| 層 | 選定 |
|----|------|
| 言語 | Node.js（ESM） |
| ブラウザ | Playwright |
| 起動 | PowerShell（`run_*.ps1`） |
| 設定 | `*.sample.json` をコピーして手元用（git しない） |
| 出力 | `output/`（gitignore） |

eBay Trading / Inventory API で価格更新する案は **Phase 5 以降の代替**として検討。まずはセルスタ運用に合わせブラウザ中心。

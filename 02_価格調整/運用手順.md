# 価格調整 班 — 運用手順

## 目標（ロードマップ Phase 2）

競合最安を踏まえた **変更案一覧** を定期的に作る。価格の **eBay 反映** は検証フェーズでは人がセルスタで行う。

## 確定事項（運用前提・2026-05 更新）

| 項目 | 内容 |
|------|------|
| 競合の「最安」 | **商品価格 + 送料の合計（total）** で比較する |
| 利益の判定 | 手数料等は **セルスタの利益計算ツールの結果を正** とする。リポジトリの `run_price_proposal` は **候補価格のたたき台** または突合用。最終の OK/NG はセルスタで確認する |
| スケール | 最初は **1 件ずつ** 検証。将来 **1 日あたり約 100 件** を目安にする（そのときはセルスタの負荷・制限に合わせて間隔・並列を設計する） |
| 仕様確認 | 応用設定・一覧画面・利益ツールの **セルスタ側仕様を先に確認** し、自動化の境界（標準機能で足りる範囲／自前が必要な範囲）を決める |
| スクショ | 共有可。**注文者名・住所・メール・トラッキング等の個人情報はマスク** してから貼る |

**手順の詳細（画面単位）**は `セルスタ実務フロー_UK基準.md`（Ship to UK・利益計算の送料$/DDP・Shipping Policy 帯の切り替え）を参照。

## ビジネスルール（固定）

| 項目 | ルール |
|------|--------|
| 競合価格 | eBay 上の最安（**商品価格 + 送料** の合計 USD）。**Ship to は United Kingdom（イギリス）** で読む（日本向け最安は使わない） |
| 目標価格 | 最安合計 **− 0.01 USD**（それ以上は下げない） |
| 最低利益 | 利益率 **1% 以上**（**還付金は含めない**）。**数値の最終確認はセルスタの利益ツールで行う** |
| 見直し | 全 SKU：**週 1 回**（PC 負荷が高いときは **月 1**）。ホット SKU：**毎日** |
| 還付金 | 試算・可否判定には使わない（参考列として CSV に出してもよい） |

## 週次サイクル（Phase 1 〜 3）

1. **対象リスト**を用意する（セルスタ出品一覧から SKU・現価・仕入・送料見積を取得）。
2. **競合最安**を記録する（手動で eBay を開く、または Phase 2 以降はスキャンスクリプト）。
3. `run_price_proposal.ps1` を実行し、`output/price_proposals_YYYYMMDD.csv` を得る。
4. CSV を確認する。
   - `action=lower` かつ `approved_by_rule=true` → 値下げ候補
   - `action=raise` → 競合より安すぎる／最安を取りつつ利益に余裕がある場合の値上げ候補
   - `action=hold` → 最安に合わせると利益不足。`floor_price_usd` までしか下げられない
   - `action=skip` → データ不足・要確認
5. **セルスタ**で価格を更新する（Phase 4 までは人がクリック）。
6. `作業ログ/YYYY-MM.md` に件数・気づき・計算式のズレを 1 行以上残す。

## 入力（`listings_input.json`）

最低限、SKU ごとに次を揃える。

| フィールド | 説明 |
|------------|------|
| `sku` | 社内 SKU |
| `sellstaId` / `ebayItemId` | 参照用（どちらか） |
| `currentPriceUsd` | 現在の出品価格（USD） |
| `shippingChargedUsd` | バイヤーへの送料（USD、無料なら 0） |
| `competitorLowestTotalUsd` | 競合の最安 **合計**（価格+送料） |
| `costJpy` | 仕入（JPY） |
| `outboundShippingJpy` | こちらが負担する国際送料見積（JPY） |
| `priority` | `hot`（毎日） / `normal`（週次） |

任意：`weightGrams`、`category`、`notes`。

## 出力（CSV 列の意味）

- `proposedPriceUsd` … 推奨の **商品価格**（送料は別枠のときは total ではない点に注意）
- `proposedTotalUsd` … 商品価格 + 自分の `shippingChargedUsd`
- `profitMarginPercent` … 提案価格での利益率
- `approved_by_rule` … 1% ルールを満たすか
- `action` … `lower` / `raise` / `hold` / `skip`
- `reason` … 人が読む短い理由

## 承認ポイント

- ルール・計算式を変えた直後：**10 SKU をセルスタの利益表示と突合**してから本番サイクル
- Phase 4 以降：自動入力後も **初回 1 週間は全件目視**またはサンプリング
- 価格差が大きい（例 **10% 超**）提案は `needs_review` として手動のみ

## 例外（自動・一括の対象外）

- 競合が自分の別アカウント・同一在庫の重複リスト
- オークション形式・バリエーション多数で最安が取れない
- セルスタ／eBay のメンテナンス・ログイン異常
- 新規出品 24 時間以内（価格を触らない運用にする場合）

## 作業ログ

`作業ログ/` に月次ファイル（例 `2026-05.md`）を置き、次を残す。

- 実行日・対象件数・提案件数（lower / raise / hold / skip）
- セルスタ標準機能の利用有無
- 失敗例（スクレイプ不可、利益計算のズレ）
